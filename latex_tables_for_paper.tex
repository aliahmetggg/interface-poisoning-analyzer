% 8. Results
% 
% Table \ref{tab:ipi-results} presents the Interface Poisoning Index scores 
% calculated for all interfaces in Apache Commons CLI. The analysis reveals 
% that 77.8\% of interfaces (7 out of 9) have only a single implementing class, 
% indicating a high prevalence of speculative abstraction.
%
% The \texttt{TypeHandler} interface exhibits the highest IPI score (0.772), 
% primarily due to its combination of single implementation, deep call chain 
% (CallDepth = 3), and 50\% unused method rate. This interface exemplifies 
% the Interface Poisoning phenomenon: it introduces unnecessary abstraction 
% layers that complicate navigation without providing practical extensibility.
%
% In contrast, the \texttt{Converter} interface demonstrates healthy interface 
% design with an IPI of 0.166. With six distinct implementations and no unused 
% methods, it represents a legitimate use of abstraction for polymorphism.
%
% The average IPI of 0.520 suggests moderate interface poisoning across the 
% project. While Apache Commons CLI is a well-maintained library, these findings 
% indicate that even mature projects accumulate unnecessary abstractions over time.

% IPI = \alpha \cdot SIR + \beta \cdot (1 - UUR) + \gamma \cdot UMR + \delta \cdot NCD

%
% where $\alpha = 0.25$, $\beta = 0.15$, $\gamma = 0.25$, and $\delta = 0.35$.
% The weights are assigned based on our research focus on navigation cost, 
% with higher emphasis on Call Depth ($\delta$) as the primary indicator 
% of developer navigation difficulty.
